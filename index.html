Values

Professional Values I believe in:
  • Write commented code to make it easy for others to understand your logic
  • Make the code as clean as possible
  • Have consistent indentation
  • Make your functions as small as possible, outsource work to new functions
  • Some design principles to follow:
    • Single Responsibility Principle - Each software entity: function, class, package, module should have one responsibility.
    • Full Responsibility Principle - Each software entity should take full responsiblity for managing its state.
    • Encapsulation Principle - restrict access to a class's data to the member functions of that class.
    • Least Knowledge Principle - Ideally a client knows only the service interface and needs to know nothing of the service implementation.
